Message Events

The Message Events API provides the means to search the raw events generated by SparkPost.

Data retention ¶

Message Events API data is retained for 10 days.

Message Events  

Search for Message Events

GET/api/v1/message-events{?bounce_classes,delimiter,events,from,message_ids,page,per_page,reason,recipients,subaccounts,template_ids,timezone,to,transmission_ids}
URI Parameters
bounce_classes

delimited list of bounce classification codes to search. (See Bounce Classification Codes.)

Example:bounce_classes=1
campaign_ids (optional, string, `Example Campaign Name`) ... delimited list of campaign ID's to search (i.e. campaign_id used during creation of a transmission).(required)  
delimiter

Specifies the delimiter for query parameter lists

Example:delimiter=:
events

delimited list of event types to search. Defaults to all event types.

Example:events=delivery, injection, bounce, delay, policy_rejection, out_of_band, open, click, generation_failure, generation_rejection, spam_complaint, list_unsubscribe, link_unsubscribe
friendly_froms (optional, list, `sender@mail.example.com`) ... delimited list of friendly_froms to search.(required)  
from

Datetime in format of YYYY-MM-DDTHH:MM.

Example:from=2014-07-20T08:00
message_ids

delimited list of message ID’s to search.

Example:message_ids=0e0d94b7-9085-4e3c-ab30-e3f2cd9c273e
page

The results page number to return. Used with per_page for paging through results.

Example:page=25
per_page

Number of results to return per page. Must be between 1 and 10,000 (inclusive).

Example:per_page=100
reason

Bounce/failure/rejection reason that will be matched using a wildcard (e.g., %reason%).

Example:reason=bounce
recipients

delimited list of recipients to search.

Example:recipients=recipient@example.com
subaccounts

delimited list of subaccount ID’s to search.

Example:subaccounts=101
template_ids

delimited list of template ID’s to search.

Example:template_ids=templ-1234
timezone

Standard timezone identification string.

Example:timezone=America/New_York
to

Datetime in format of YYYY-MM-DDTHH:MM.

Example:to=2014-07-20T09:00
transmission_ids

delimited list of transmission ID’s to search (i.e. id generated during creation of a transmission).

Example:transmission_ids=65832150921904138

Perform a filtered search for message event data. The response is sorted by descending timestamp.

Example

Request

GET /api/v1/message-events?bounce_classes=1&delimiter=:&events=delivery, injection, bounce, delay, policy_rejection, out_of_band, open, click, generation_failure, generation_rejection, spam_complaint, list_unsubscribe, link_unsubscribe&from=2014-07-20T08:00&message_ids=0e0d94b7-9085-4e3c-ab30-e3f2cd9c273e&page=25&per_page=100&reason=bounce&recipients=recipient@example.com&subaccounts=101&template_ids=templ-1234&timezone=America/New_York&to=2014-07-20T09:00&transmission_ids=65832150921904138
